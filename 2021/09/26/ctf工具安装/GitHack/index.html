GitHack安装与使用

注：python3之前已配置好，不再多讲。应该基本同于python2。

简介：

GitHack是一个.git泄露利用脚本，通过泄露的.git文件夹下的文件，重建还原工程源代码。渗透测试人员、攻击者，可以进一步审计代码，挖掘：文件上传，SQL注射等安全漏洞。

问题：

网上下载到的githack基本只适用于python2。而目前大部分使用的python版本都是python3。

方案一：安装python2来运行githack

下载python2和python3

官网https://www.python.org/下载python2和python3或者应用商城中下载。安装建议一路默认next下去。

配置python2.x的环境变量

需要将两个目录添加到path环境变量中：

  • python命令文件所在目录： python 2.x安装目录
  • pip、easy_install等其他脚本文件所在目录： python 2.x安装目录下的Scripts目录

找到此电脑选择属性或者右键我的电脑找到属性——>高级系统设置——>环境变量——>用户变量下的PATH——>编辑

将下面这两行内容添加到Path变量的开始位置：（先新建后上移到第一行）
C:\Python27
C:\Python27\Scripts\

将python 2.x 和 python 3.x的相关命令区分开来

找到 python2 和 python3 的安装目录，修改 python2.x 和 python3.x 中 python.exe 的名称为 python2.exe和 python3.exe。

然后运行cmd命令，输入python2 可以运行 python2.x 版本，输入 python3 即可运行 python3.x 版本。如下图。

解决两者pip共存问题

Python安装包需要用到包管理工具pip，但当同时安装python2和python3的时候，pip只是其中一个版本。为了保证共存，需要分别重新安装pip。

1.安装pip2：在命令窗口中输入：python2 -m pip install –upgrade pip –force-reinstall，如下图显示重新安装成功。

2.安装 pip3： 在命令窗口中输入：python3 -m pip install –upgrade pip –force-reinstall 。如果没有成功则可能在之前已经重新安装过pip.exe了，需要将它从文件夹中删除再输入命令。如下图显示重新安装成功。

3.现在可以通过pip2 -V 和 pip3-V 查看两个版本的pip信息，以后只需运行 pip2 install XXX 和 pip3 install XXX 即可安装各自的python包。到这里 pip共存问题 也已经解决啦！

使用GitHack

在含有GitHack.py的文件夹的地址栏中输入cmd打开命令窗口

输入命令：python2 GitHack.py URL.git来使用githack
